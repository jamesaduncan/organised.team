Organised Team

This is a placeholder for now but Organised Team provides a small number of microdata schemas that are used to describe various elements of the Organised Team platform. The schemas are designed to be used in exchanging data with organised team, and other software platforms. It builds upon Schema.org and extends it with additional types and properties that are specific to Organised Team.

The following schemas are documented:

  • Policy - A schema for policy definitions
  • Team - Represents a team as an organization
  • UserNeed - A schema that defines an expectation a user has
  • Schema - Defines a schema for Schemas
  • Property - A schema for schema properties
  • Enumerated - Defines a datatype that allows for a fixed set of values
  • Cardinal - The Cardinal datatype is an Enumerated type that indicates the cardinality of a property in a schema.
  • Organization - A definition for organisations that have and fulfil needs
  • Schema - A schema for a standard definition
  • SkillsFramework - defines the structure for a professional skills framework
  • Role - defines a specific job role or position within a skills framework
  • Skill - an individual competency or ability within a skills framework
