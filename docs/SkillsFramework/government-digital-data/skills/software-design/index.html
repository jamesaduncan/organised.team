  • Software Design

    • Software design involves creating the structure, components, and interfaces of software systems to meet functional and non-functional requirements while ensuring maintainability and scalability.

    Proficiency Levels

    1. Awareness

      You can understand software design principles and recognise the importance of good design in creating maintainable and scalable government software systems.

    2. Working

      You can contribute to software design activities, apply basic design patterns, and create simple software designs following established standards and guidelines.

    3. Practitioner

      You can design complex software systems, apply advanced design patterns, and establish design standards and practices for development teams.

    4. Expert

      You can lead software design strategy, establish design frameworks and methodologies, and mentor others in advanced software design techniques.
